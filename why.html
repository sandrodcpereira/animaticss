What are frame animations?

Frame animations, as the name implies, have separate frames that, when played in sequence, give the illusion of movement — pretty much like traditional animation from old cartoons.

For Animaticss, animations frames are composed into a long horizontal strip, like film in an old movie.

Why use frame animations?

If you’re looking to some reasons to use frame animations like these, here’s a few:

  • It’s a much more efficient way of storing animations compared to the GIF format.

  • It can be achieved with simple HTML and CSS. This means it has no dependencies — it’s as lightweight as your asset file.

  • You have a lot of control over when the animation plays. You can trigger it only when specific events occur, like on hover or even a button press. See examples here.

  • Only requires one single asset per animation. That’s a lot easier than handling and having to preload multiple assets.

  • Additionally, a single asset can contain multiple animations with some more complex code! Again, check the first example here.
